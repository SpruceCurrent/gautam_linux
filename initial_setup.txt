# Check that the 'gautam_linux' directory exists in the current directory

# Check all links that will be overwritten

# Check that the 'bundle' directory is empty

Make links:
    ln -s gautam_linux/home/gautam/dot_bashrc    ~/.bashrc
    ln -s gautam_linux/home/gautam/dot_vim       ~/.vim
    ln -s gautam_linux/home/gautam/dot_vim/vimrc ~/.vimrc

To use Google's public DNS servers
    sudo ln -s gautam_linux/misc/resolv_conf_head /etc/resolvconf/resolv.conf.d/head

If using xmonad:
    ln -s gautam_linux/home/gautam/dot_xmonad ~/.xmonad
    ln -s gautam_linux/home/gautam/dot_xmobarrc ~/.xmobarrc

If using i3:
    ln -s gautam_linux/i3/config ~/.i3/config

For using the compose key and defining custom key-mappings
(haven't yet figured out how to make it work with xmonad)
    ln -s gautam_linux/home/gautam/dot_XCompose ~/.XCompose
    ln -s gautam_linux/home/gautam/dot_gnomerc ~/.gnomerc

Clone Vundle:
    git clone https://github.com/gmarik/vundle.git ~/.vim/bundle/vundle

Install all Vim plugins using Vundle:
    vim +PluginInstall +qall

SSH config file:
    ln -s gautam_linux/misc/ssh_config ~/.ssh/config

Other convenience links:
    sudo ln -s /usr/bin/google-chrome /usr/bin/chrome

    # Link 'todo.sh' to a pre-existing location in PATH so that it can be directly launched using
    # dmenu.
    sudo ln -s /home/gautam/gautam_linux/scripts/todo.sh /usr/local/sbin/todo

    # Backup existing dmenu_path before doing this?
    sudo ln -s /home/gautam/gautam_linux/scripts/dmenu_path.sh /usr/bin/dmenu_path
